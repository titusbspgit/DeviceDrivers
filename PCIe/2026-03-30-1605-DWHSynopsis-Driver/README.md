# DWHSynopsis (Synopsys DWC PCIe) Driver

Generated by RCI-ag-DrvGen Agent.

- Arch: ARM (bare metal), Compiler: gcc
- Endianness: CPU big-endian; device registers assumed little-endian (configurable)
- Features:
  - Link control (bring-up, status, directed speed change)
  - iATU (outbound region) programming via viewport model
  - MSI host block (vector enable/mask/status)

## Notes and Assumptions
- DBI base address is provided by platform integration (SoC-specific) and passed to dw_pcie_init().
- iATU programming model assumed to be the Viewport model. If Unroll model is present/desired, extend the iATU driver accordingly.
- Some SoCs expose LNKCTL2.DIRECTED_SPEED_CHANGE; code sets it if the macro is defined. By default the macro is 0 (no-op), and Port Logic GEN2_CTRL.DIRECTED_SPEED_CHANGE is always used.

## Endianness Handling
- CPU is big-endian while the DWC DBI is little-endian; HAL converts on 32-bit paths and computes correct sub-word lanes for 8/16-bit operations to avoid misaligned or incorrect byte selection.
